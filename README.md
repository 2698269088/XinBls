# XinBls 插件

XinBls 是一个 XinBot 机器人插件，专门用于记录游戏中的各种活动和聊天信息到本地日志文件中。

## 功能特性

### 日志记录
- 自动记录玩家加入/离开游戏的信息
- 记录玩家聊天消息
- 记录系统通知和其他重要事件

### 日志管理
- **自动轮转**：当日志文件达到指定行数时，自动创建新文件
- **自动清理**：可配置是否自动清理旧日志文件
- **ANSI 颜色过滤**：自动移除终端颜色代码，避免在文本编辑器中显示乱码
- **智能过滤**：可选择性地跳过包含特定关键词的消息

### 配置管理
- 支持 JSON 格式的配置文件
- 可动态调整各项参数
- 配置文件位于日志目录下 (`./plugin/XinBls/config.json`)

## 配置选项

配置文件 `config.json` 包含以下可配置参数：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `maxLogLines` | 整数 | 20000 | 单个日志文件的最大行数，超过此数值自动创建新文件 |
| `autoCleanEnabled` | 布尔值 | true | 是否启用自动清理功能 |
| `maxLogFileCount` | 整数 | 10 | 最大日志文件数量，超过此数量会清理最旧的文件 |
| `skipMessagesWithFrom` | 布尔值 | false | 是否跳过包含"来自"的消息 |
| `skipMessagesWithTo` | 布尔值 | false | 是否跳过包含"发至"的消息 |

## 安装说明

1. 将插件文件放入 Minecraft 机器人的插件目录
2. 启动机器人，插件将自动创建配置文件和日志目录
3. 根据需要编辑 `./plugin/XinBls/config.json` 文件调整配置
4. 重启机器人使配置生效

## 日志文件位置

日志文件存储在 `./plugin/XinBls/` 目录下，文件命名格式为 `xinbls-YYYY-MM-DD-HHMMSS.log`

## 使用示例

默认配置下，插件会在以下情况执行相应操作：
- 当日志文件达到 20000 行时，自动创建新文件
- 如果启用了自动清理且日志文件数量超过 10 个，会自动删除最旧的日志文件
- 如果启用了过滤功能，包含"来自"或"发至"的消息将不会被记录

## 开发

本插件基于 xinbot 框架开发，使用 SLF4J 和 Logback 进行日志管理，使用 Gson 处理 JSON 配置。
